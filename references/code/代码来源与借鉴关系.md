## 2. Reference 代码目录说明

### 2.1 `difflifting/`



### 2.2 `topobench/`


## 3. 完整代码库和实际借鉴范围

本包已经包含完整 TopoBench 和 DiffLifting 代码库，但本方案并不是要求学生整体移植它们。

实际借鉴范围如下：

### 3.1 DiffLifting 实际借鉴范围

完整代码库：

```text
references/full_repos/difflifting-main/difflifting-main
```

实际重点文件：

```text
model/tnn_with_lifting_graph_classific.py
model/tnn_with_lifiting.py
model/TNN.py
```

实际借鉴点：

```text
1. 用 GNN/GPS 得到 node latent embedding
2. 用 MLP 对候选 higher-order cells 输出纳入概率
3. 用 Gumbel-softmax / STE 保持选择过程可反传
4. 将 learned lifting 后的 incidence 交给 TNN
```

本方案落地到 RAGraph 时，只移植第 2 和第 3 点：

```text
DiffLiftRingSelector.py
```

它只对 RAGraph 已有的 ring/2-cell candidates 做可学习选择，不重写完整 incidence/laplacian/TNN pipeline。

### 3.2 TopoBench / TopoTune 实际借鉴范围

完整代码库：

```text
references/full_repos/topobench-main/topobench-main
```

实际重点文件：

```text
topobench/nn/backbones/combinatorial/gccn.py
topobench/nn/backbones/graph/gps.py
configs/model/cell/topotune.yaml
configs/model/graph/gps.yaml
```

实际借鉴点：

```text
1. combinatorial complex 可以展开为不同 rank/route 的 Hasse graph
2. rank-aware neighborhood route 比单纯 graph adjacency 更适合表达 cell complex
3. Transformer/GPS 是可选 backbone，不一定稳定优于 GCN/GIN
```

当前落地代码没有直接移植 TopoTune，因为第一阶段重点是 memory/retrieval。TopoTune 主要用于后续 Hasse reasoning 消融。

### 3.3 RAGraph 实际借鉴范围

原始项目不整体打包，因为包含较多数据、模型和历史目录。当前包只保留最小对照：

```text
references/code/ragraph_original/RAGraph2.py
references/code/ragraph_original/extract_ring.py
```

实际借鉴点：

```text
1. 保留 RAGraph2 的 forward / forward_with_loss 接口
2. 保留 ToyGraphBase 的 resource key/value/label 构建方式
3. 保留 ring contrastive learning 的基本 CTCL 目标
```

## 4. 为什么仍保留关键代码快照

虽然已经包含完整代码库，但完整仓库较大且层级复杂。`references/code` 中的快照是给学生快速阅读用的最小入口。

因此本包只保留：

```text
能解释方法来源的最小 reference code
能直接复制到 RAGraph 的新增代码
能指导实验的文档和脚本模板
```

这样包小、路径清楚、学生不容易被大仓库干扰。
