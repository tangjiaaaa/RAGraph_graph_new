# 借鉴论文清单与阅读顺序

这份清单用于解释本方案中每个模块的论文来源。学生不需要先读完所有论文，建议按优先级阅读。

## 1. 必读论文

### 1.1 RAGraph

论文：

```text
Retrieval-Augmented Graph Learning
NeurIPS 2024
```

链接：

```text
https://proceedings.neurips.cc/paper_files/paper/2024/hash/34d6c7090bc5af0b96aeaf92fa074899-Abstract-Conference.html
https://arxiv.org/abs/2410.23855
```

和本项目的关系：

- 这是原始代码 `RAGraph-new` 的主要思想来源。
- 原模型的 toy graph library、retrieval、graph prompt/fusion 都来自这条线。
- 本方案不是推翻 RAGraph，而是在其 retrieval memory 上加入 cellular topology、learned lifting 和 task-aware rerank。

对应参考代码：

```text
references/code/ragraph_original/RAGraph2.py
references/code/ragraph_original/extract_ring.py
```

## 1.2 DiffLifting

论文：

```text
Differentiable Lifting for Topological Neural Networks
ICLR 2026
```

链接：

```text
https://openreview.net/forum?id=eC89CbINIw
```

和本项目的关系：

- DiffLifting 的核心不是静态构造更多 cell，而是学习哪些候选高阶结构应被纳入。
- 本方案借鉴它的思想，但没有完整移植其 KNN/cell incidence 构造。
- 我们采用更小的可落地版本：在 ReTAG 已有 ring/2-cell candidates 上学习 selector。

对应新增模块：

```text
code_to_copy/RAGraph_graph_new/ragraph_utils/DiffLiftRingSelector.py
```

对应参考代码：

```text
references/code/difflifting/tnn_with_lifting_graph_classific.py
references/code/difflifting/tnn_with_lifiting.py
references/code/difflifting/TNN.py
```

重点看参考代码中的：

```text
compute_node_cell_matrix
k_mlp
edge_mlp
F.gumbel_softmax
incidence_2
```

## 1.3 TopoTune

论文：

```text
TopoTune: A Framework for Generalized Combinatorial Complex Neural Networks
ICML 2025
```

链接：

```text
https://openreview.net/forum?id=S5njonQdBf
https://geometric-intelligence.github.io/topotune/
```

和本项目的关系：

- TopoTune 提供的是 combinatorial complex 上的 route/Hasse 展开视角。
- 它的代码不是简单“cell complex transformer”，而是把不同 rank 的邻域关系展开成图，再用 GCN/GIN/GAT 等基础 GNN。
- 本方案当前主线没有强行加入 TopoTune 层，因为最先要验证 retrieval/memory 是否有效。
- 后续可以用 TopoTune/Hasse-style encoder 做 S5 消融。

对应参考代码：

```text
references/code/topobench/gccn.py
references/code/topobench/gps.py
references/code/topobench/topotune_cell.yaml
references/code/topobench/gps_graph.yaml
```

重点看：

```text
TopoTune.forward
intrarank_expand
interrank_expand
aggregate_inter_nbhd
configs/model/cell/topotune.yaml
```

## 2. 选读论文

### 2.1 Task-Aware Retrieval Augmentation for Dynamic Recommendation

论文：

```text
Task-Aware Retrieval Augmentation for Dynamic Recommendation
```

链接：

```text
https://arxiv.org/html/2511.12495
```

和本项目的关系：

- 该论文没有可用官方代码。
- 主要借鉴点不是 Graph Transformer 本身，而是 task-aware retrieval / reranking 的思路。
- 本方案中的 `TaskAwareRetriever.py` 是它的轻量可落地替代：先按原 RAGraph 相似度召回，再根据 query-candidate interaction 重新加权。

对应新增模块：

```text
code_to_copy/RAGraph_graph_new/ragraph_utils/TaskAwareRetriever.py
```

## 2.2 Reflective Memory Management

论文：

```text
Reflective Memory Management
ACL 2025
```

链接：

```text
https://aclanthology.org/2025.acl-long.413/
https://arxiv.org/abs/2503.08026
```

和本项目的关系：

- 它是 agent/LLM memory 论文，不是 graph learning 论文。
- 本方案只借鉴“根据后续反馈校准 memory utility”的思想。
- 不应把本模型写成 agent memory。

建议论文表述：

```text
Inspired by reflective memory management, we introduce supervised reflective utility calibration for cellular memories in graph representation learning.
```

论文：

```text
MemMap: Adaptive and Latent Memory Structure for Dynamic Graph Learning
KDD 2024

Mayfly: A Neural Data Structure for Graph Stream Summarization
ICLR 2024 Spotlight
```

链接：

```text
https://kdd2024.kdd.org/research-track-papers/
```

## 3. 推荐阅读顺序

1. 先读 `RAGraph2.py`，理解原模型如何 retrieve。
2. 读 `references/code/ragraph_original/extract_ring.py`，理解 ring feature 怎么来。
3. 读 DiffLifting README 和 `compute_node_cell_matrix`，理解 learned lifting。
4. 读 `DiffLiftRingSelector.py`，看我们如何把 DiffLifting 思想落到 ReTAG。
5. 读 `TaskAwareRetriever.py`，理解 rerank。
6. 读 `ToyGraphBase.py` 的 `resource_utility` 和 `update_memory_utility`，理解 reflective memory calibration。
7. 最后读 TopoTune 的 `gccn.py`，作为后续 Hasse reasoning 消融。
